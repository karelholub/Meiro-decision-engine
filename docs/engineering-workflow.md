# Engineering Workflow

## Obsidian Usage

Use the local Obsidian vault at `/Users/kh/paperclip` for engineering knowledge that should survive beyond a single task or chat thread.

Write daily notes for timeline-oriented work:

- start/end context for a work session
- quick observations from debugging or reviews
- meeting notes, decisions in progress, and handoff reminders
- raw notes that may need cleanup later

Write PARA memory entries for durable knowledge:

- stable project facts, architecture decisions, and recurring operational rules
- reusable how-to guidance, runbooks, and investigation outcomes
- active project material that should be grouped under Projects, Areas, Resources, or Archives
- synthesized lessons from daily notes that are likely to be reused

Default rule: capture fast-moving context in a daily note first, then promote it to a PARA entry when it becomes reusable or authoritative.
